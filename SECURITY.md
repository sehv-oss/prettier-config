# Security Policy

We take the security of this project seriously and appreciate responsible disclosure of vulnerabilities.

## Supported Versions

Unless stated otherwise in the project's README, only the **latest release** receives security updates.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public issues, discussions, or pull requests.**

Instead, report them privately using one of the following channels:

1. **GitHub Private Vulnerability Reporting** (preferred): go to the **Security** tab of this repository and click **"Report a vulnerability"**.
2. **Email**: send the details to <sehnemvinicius@gmail.com> with the subject line `[SECURITY] <repository name>`.

### What to Include

To help us understand and resolve the issue quickly, please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue, or a proof of concept
- Affected versions, configurations, or environments
- Any suggested mitigation or fix, if you have one

### What to Expect

- We will acknowledge your report within a reasonable time frame
- We will keep you informed of the progress toward a fix
- We will credit you in the disclosure, unless you prefer to remain anonymous
- We ask that you give us a reasonable amount of time to address the issue before any public disclosure

Thank you for helping keep this project and its users safe! 🔒
